# 面试讲解稿

## 一句话介绍

这是一个自动驾驶跟车制动场景仿真与离线分析原型。项目用 C++ 做简化车辆纵向仿真，输出 CSV 日志，再用 Python 读取日志并绘制速度、加速度、车距和 TTC 曲线。

## 项目定位

这个项目不是完整自动驾驶算法，也不是量产级仿真平台。它的定位是自动驾驶测试方向的小型练习项目，重点是理解测试场景、日志输出和离线分析流程。

## 为什么做这个项目

自动驾驶仿真测试和路测测试中，经常需要关注跟车、前车制动、车距过近、TTC 过低、纵向减速度过大等问题。这个项目用简化方式模拟这些数据产生和分析流程。

## C++ 部分做了什么

- `Vehicle` 表示自车，保存位置、速度和加速度。
- `LeadVehicle` 表示前车，支持在指定时间开始制动。
- `Controller` 表示简单跟车规则，根据车距和 TTC 决定是否轻刹或重刹。
- `Logger` 负责把每一步仿真数据写入 CSV。
- `main.cpp` 负责创建对象、推进仿真循环、计算车距和 TTC、写入日志。

## CSV 字段怎么理解

- `time`：仿真时间。
- `ego_position`、`ego_speed`、`ego_acceleration`：自车状态。
- `lead_position`、`lead_speed`、`lead_acceleration`：前车状态。
- `distance`：前车位置减去自车位置。
- `throttle`：简化油门指令。
- `brake`：简化制动指令。
- `ttc`：简化碰撞时间指标。

## TTC 怎么算

当前项目使用简化 TTC：

```text
relativeVelocity = ego_speed - lead_speed
```

如果 `relativeVelocity <= 0`，说明自车没有追近前车，TTC 记为 999。

如果 `relativeVelocity > 0`，使用：

```text
TTC = distance / relativeVelocity
```

TTC 越小，说明追尾风险越高。

## Python 部分做了什么

Python 脚本使用 pandas 读取 `data/log.csv`，使用 matplotlib 生成三张图：

- 自车和前车速度曲线
- 自车和前车加速度曲线
- 车距和 TTC 曲线

## 面试中要避免夸大

不要说：

- 实现了真实自动驾驶系统
- 实现了真实 AEB 算法
- 处理了真实路测数据
- 使用了 ROS 或 Carla
- 支持批量多场景自动评测

可以说：

- 实现了一个简化的跟车制动场景仿真原型
- 能输出 CSV 日志
- 能用 Python 对日志做基础可视化分析
- 能通过车距和 TTC 观察风险变化

## 一分钟讲解版本

这个项目是一个自动驾驶测试方向的小型仿真与离线分析原型。我用 C++17 实现了自车、前车、简单跟车控制器和 CSV 日志模块，构造前车制动场景，并在每个仿真步输出速度、加速度、车距、制动状态和 TTC 等字段。然后用 Python 读取 CSV 日志，绘制速度、加速度、车距和 TTC 曲线。项目重点不是复杂自动驾驶算法，而是测试场景构造、日志记录和离线分析流程。
